Final Project Proposal

For my final project, I plan to build a data visualizer that enables users to explore Strava aggregated Strava data to view trends between different activities over differentiating timeperiods.

Stage Descriptions

Projected Development Process

  1. Research and make sense of the Strava API. Map out what data I will need to access from the API, how many calls to the API I will need to make, and the parameters on which those calls will be made
  2. Make datasets from API calls that will be used by the data visualization
  3. Make the graphs chart interface with the different buttons/areas for user input to change the displayed data.
  4. Program the user input to change the collected data
  5. Display the data
  6. Prototype Version
  7. Enable the change from one display to another display
  8. Add details such as labels
  9. Make the details/labels responsive to the changes in user input
  10. Explore adding a third dimension to the user access to data
  11. Final Version

Unknowns

  • What and how to get the right data from the API
  • How to efficiently make many calls to an API
  • How to make new datasets with the API response
  • How to make a query based off user input
  • How to configure the user input options

Stretch Goals

Some stretch goals for this project are:

  • Building new tables of data from sequenced and specific API calls
  • Using Chart.js and d3.js more comprehensively
  • Implementing different types of data visualizations rather than just bar graphs. I would like to explore representing differing averages of values in pie charts and other graphs.
  • Designing how the user will interact with the interface to make specific queries
