Final Project Proposal

For my final project, I plan to build a data visualizer that enables users to explore aggregated Strava data to view trends between different activities over differentiating time periods.

Two envisioned interfaces for the data visualization, with the one on the left representing a less-developed interface and the one on the right a more developed interface.

Stage Descriptions

Prototype

For my prototype version, my main goal is having my API data handling mostly configured so that the user can select between a few different graphs of pre-generated queries and the respective data from the API response. This version will be effective at establishing that I am correctly collecting and orgnaizing the API data and cleaning it so it can be displayed. This version will also showcase that I have the visualization interface up and running

Final Version

The final version will be where I implement the user interaction features. I would like to incorporate dropdown menus that will allow the user to choose between different activities' data for viewing, different types of charts for the data to be represented in, and different time periods to collect over. My hope is to enable the user to choose from at least three different options in each category, potentially more in the "activity" category. I would like the data visualization to change on user interaction events so that the change is instant rather than the user having to press a button to create a new graph.

Projected Development Process

  1. Research and make sense of the Strava API. Map out what data I will need to access from the API, how many calls to the API I will need to make, and the parameters on which those calls will be made
  2. Make datasets from API calls that will be used by the data visualization
  3. Make the graphs chart interface with the different buttons/areas for user input to change the displayed data.
  4. Program the user input to change the collected data
  5. Display the data
  6. Prototype Version
  7. Enable the change from one display to another display
  8. Add details such as labels
  9. Make the details/labels responsive to the changes in user input
  10. Explore adding a third dimension to the user access to data
  11. Final Version

Unknowns

  • What and how to get the right data from the API
  • How to efficiently make many calls to an API
  • How to make new datasets with the API response
  • How to make a query based off user input
  • How to configure the user input options

Stretch Goals

Some stretch goals for this project are:

  • Building new tables of data from sequenced and specific API calls
  • Using Chart.js and d3.js more comprehensively
  • Implementing different types of data visualizations rather than just bar graphs. I would like to explore representing differing averages of values in pie charts and other graphs.
  • Designing how the user will interact with the interface to make specific queries
